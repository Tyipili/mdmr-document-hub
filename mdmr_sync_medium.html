1. CHANNEL CONFIGURATION SET PARAMETER

Defines the channel mapping and configuration for meter data collection
Attribute Value
TIS Reference Section 2.2.6.5, Table 2.2.6.5-1
Parameter Type Meter Parameter (configuration-specific)
Future Dating ✗ NOT ALLOWED - Must change with meter
Mandatory YES - Required for interval meter processing

When Changes Are Required (Business Triggers)

  • Physical meter change to meter with different channel configuration
  • Meter configuration update requiring different channel mappings
  • Change in metering requirements (e.g., adding demand channels)
  • Net metering installation or removal
  • Advanced metering feature activation

⚠ CRITICAL RULE: Channel Config Changes Require Meter Changes

Channel Configuration Set can only be changed when performing a physical or logical meter change. You cannot update this parameter in isolation - it must coincide with ending the old meter relationship and starting a new one.

Common Channel Configuration Sets

Configuration Set Description
Standard Residential Single channel for consumption (kWh), typically for residential customers
Small General Service Consumption (kWh) with optional demand channels
Large General Service Multiple channels including consumption, demand (kW), and power factor
Net Metering Separate channels for delivered and received energy
Time-of-Use (TOU) Separate channels for on-peak and off-peak periods

Synchronization Process

Meter Change with Different Channel Config

Scenario B (2 Syncs Required):

  1. Sync 1: End old meter with old channel config
  2. Sync 2: Start new meter with new channel config

This two-sync approach ensures clean data separation between different channel configurations.

Required Synchronization Files & Fields

File Field/Element Notes
File 01 Meter Asset with Channel Config Set Specify the channel configuration set code
File 05 Meter Relationships End old meter, start new meter with new config

2. NET METERING CONFIGURATION

Special configuration for bidirectional energy measurement in net metering scenarios

What is Net Metering?

Net metering allows customers with generation facilities (e.g., solar panels) to receive credit for excess electricity delivered back to the grid. The meter must track both delivered energy (from grid) and received energy (to grid).

Net Metering Requirements

Component Requirement
Channel Configuration Must use approved net metering channel config set
Framing Structure Must use compatible framing structure (certain framing structures support net metering)
Meter Capability Meter must support bidirectional measurement
Data Channels Separate channels for delivered and received energy

⚠ Compatibility Check Required

Not all Channel Configuration Sets are compatible with all Framing Structures. Verify compatibility before submitting synchronization, especially for net metering scenarios. Reference TIS Section 2.3.10 for compatibility matrix.

Net Metering Setup Process

  1. Verify customer eligibility for net metering program
  2. Select appropriate net metering channel configuration set
  3. Verify framing structure compatibility
  4. Perform meter change (if existing meter doesn't support net metering)
  5. Update channel configuration in synchronization
  6. Verify proper data flow for both delivered and received channels

3. INTERVAL LENGTH PARAMETER

Defines the time interval for meter data collection (e.g., 15-minute, 60-minute)
Attribute Value
TIS Reference Section 2.2.3, Meter Asset Configuration
Parameter Type Meter Configuration (hardware-specific)
Future Dating ✗ NOT ALLOWED - Must change with meter
Mandatory YES - Required for all interval meters

Common Interval Lengths

Interval Length Typical Usage Data Volume
15 minutes Large general service, detailed load profiling 96 intervals per day
60 minutes Most common for residential and small commercial 24 intervals per day
5 minutes Specialized applications, high-resolution monitoring 288 intervals per day

⚠ CRITICAL: Interval Length Must Match Physical Meter

The Interval Length specified in synchronization MUST exactly match what the physical meter is configured to report. A mismatch will cause meter read processing failures and Business Rule violations (Business Rule 25).

Changing Interval Length

Like Channel Configuration Set, Interval Length can only be changed during a meter change operation:

  • Perform physical meter change to meter with different interval length
  • Use Scenario B (2 syncs) to properly end old configuration and start new
  • Ensure clean data separation between different interval lengths
  • Verify AMCC is configured to expect new interval length

4. ADVANCED PARAMETERS

Additional specialized parameters for specific metering scenarios

Load Profile Configuration

Controls how meter data is collected, stored, and transmitted to the MDM/R for load profiling purposes.

Demand Configuration

Parameter Description
Demand Window Time period for demand calculation (e.g., 15-minute block)
Demand Reset How often demand values are reset (monthly, billing period)
Peak Demand Tracking Whether to track and report peak demand occurrences

Power Quality Parameters

  • Voltage monitoring thresholds
  • Power factor measurement configuration
  • Harmonic distortion tracking
  • Event logging parameters

⚠ Advanced Parameters Require Specialized Knowledge

These parameters typically require consultation with metering engineering staff and careful validation before synchronization. Incorrect configuration can impact data quality and billing accuracy.

5. RELATED REPORTS REFERENCE

Understanding the synchronization report outputs for successful implementation

IR06 - Business Rule Exception Report

Attribute Details
Purpose Lists synchronization submissions that violate MDM/R business rules
Key Rules for Medium Priority Business Rule 25 (Interval Length), Business Rule 18 (Meter Reuse), Business Rule 20 (Relationship Management)
Action Required Review exceptions, correct data, resubmit synchronization

IR07 - Synchronization Processing Exception Report

Attribute Details
Purpose Provides detailed information on exceptions during synchronization processing
When Generated After staging table loader completes, during business rule validation
TIS Reference Section 2.3.2.4

IR14 - Staging Table Loader Exception Report

Attribute Details
Purpose Reports file validation errors before synchronization processing begins
Check First Always review IR14 first - if it has errors, IR06 and IR07 may not be generated
Common Issues Formatting errors, invalid codes, missing required fields, data type mismatches

QUICK TIPS FOR SUCCESS

  • Channel Configuration Sets can only change with meter changes - never in isolation
  • Verify Channel Config Set and Framing Structure compatibility before synchronization
  • Interval Length must exactly match the physical meter configuration
  • Net metering requires specific channel configurations and compatible framing structures
  • Use Scenario B (2 syncs) when changing channel configuration or interval length
  • Always check IR14 report first for validation errors
  • Review Business Rules 18, 20, and 25 for meter and relationship management
  • Test configuration changes in non-production environment when available
  • Document all configuration changes for future reference and troubleshooting
  • Consult with metering engineering for advanced parameter configurations
