1. OVERVIEW

This guide provides comprehensive instructions for synchronizing a brand new Service Delivery Point (SDP) with a new meter installation to the MDM/R system.

Scenario A Definition:

  • The SDP has never been synchronized to MDM/R before
  • This is the first meter ever at this location
  • Universal SDP ID was just assigned
  • Complete new setup required

When to Use This Guide

Use this guide when:

  • Installing meter at a brand new location (new construction)
  • First-time synchronization of an SDP to MDM/R
  • Setting up all mandatory elements from scratch
  • No previous meter history exists at this SDP

Not Covered in This Guide:

  • Meter changes (replacing existing meter) - see Meter Change Guide
  • Updates to existing SDP parameters
  • Meter removals without replacement

2. PRE-REQUISITES

Before You Begin

Ensure you have all the following information and setup completed before creating synchronization files:

Site Information

  • Universal SDP ID assigned (10-character)
  • Physical address verified
  • Premise details (postal code, municipal ID if applicable)
  • Geocode coordinates (if available)

Meter Information

  • Meter ID (unique identifier)
  • Meter type and manufacturer
  • Interval length (15-minute, 60-minute, etc.)
  • Channel configuration set
  • Number of dials
  • Meter installation date/time (in EST)

Communication Module Information

  • Communication module ID (AMCD ID)
  • Module type
  • Installation date/time (in EST)

Business Information

  • Customer rate class determined
  • Commodity type identified (electricity)
  • Framing structure determined
  • Billing Agent and AMI Operator ORG IDs known

⚠ CRITICAL: Date/Time Requirements

All date/time values in synchronization files MUST be in Eastern Standard Time (EST), not Eastern Daylight Time (EDT). This applies year-round, even during daylight saving time periods.

Field Value/Options Notes

Address [Street address] Physical location

3.3 File 03: Service Agreement Data File

Field Value/Options Notes
Record Indicator Service Agreement Required
Universal SDP ID [Your SDP ID] Must match other files
Framing Structure Code from approved list Determines billing framing
Start Date/Time [YYYYMMDDHHMMSS in EST] When framing becomes effective
Commodity Type Electricity Always "Electricity"

⚠ Framing Structure Midnight Rule

Framing Structure changes must be synchronized with Start Date/Time at midnight EST (00:00:00) of the effective date. This ensures proper daily framing boundaries.

3.4 File 04: Parameter Data File

Element 1: VEE Service Parameter

Field Description
Parameter Type VEE Service
Universal SDP ID Links parameter to SDP
VEE Service Code Defines validation/estimation rules (e.g., RES01 for residential)
Start Date/Time When VEE service becomes effective (typically meter install time)

Element 2: Dials Parameter

Field Description
Parameter Type Meter Parameter - Dials
UDC ID Meter ID this applies to
Number of Dials Must match physical meter (e.g., 5 or 6 dials)
Start Date/Time Same as meter installation time

⚠ CRITICAL: Dials Must Match Meter

The number of dials specified MUST exactly match what the physical meter reports. Mismatch causes Business Rule 25b violations and VEE processing errors.

4. OPTIONAL ELEMENTS

While not mandatory for producing billing quantities, these elements may be required based on your business processes:

CT/PT Multiplier (File 04)

Required only for meters with current transformers (CT) or potential transformers (PT):

  • Most residential meters: No CT/PT required (multiplier = 1)
  • Large commercial/industrial: CT/PT multiplier required
  • Must start at same time as SDP to Meter relationship

Default CT/PT Behavior

If no CT/PT multiplier is provided, MDM/R assumes unity (1.0). Only provide CT/PT parameter when the multiplier is NOT equal to 1.

Distributor Rate Class - DRC (File 04)

May be required depending on settlement and billing requirements.

Commodity Rate Class - CRR (File 04)

Required for retail energy contract customers.

SDP to Energy Service Provider (File 05)

For customers with retail energy contracts.

SDP to CCA Service Provider (File 05)

For Community Choice Aggregation services.

5. COMPLETE FILE SET STRUCTURE

Required Files (All 7 Must Be Present)

File # File Name Required Content
00 Manifest File File Name Record + Header + List of all files
01 Asset Data File File Name Record + Header + Detail Records (SDP, Meter, Comm Module)
02 Premise Data File File Name Record + Header + Detail Records (Premise data)
03 Service Agreement Data File File Name Record + Header + Detail Records (Framing Structure)
04 Parameter Data File File Name Record + Header + Detail Records (VEE Service, Dials, etc.)
05 Relationship Data File File Name Record + Header + Detail Records (All relationships)
07 Component SDP File File Name Record + Header only (not loaded by MDM/R)

Important: File Set Completeness

All six mandatory files (plus Component SDP) must be present in the sync set. Even if a file has no detail records, it must contain File Name Record and Header Record.

Record Count Summary

Setup Level Elements Required Estimated Records
Minimum Mandatory All required for billing quantities ~15-20 records
Typical Residential Mandatory + common optional ~20-25 records
Commercial with CT/PT Mandatory + CT/PT + DRC ~25-30 records

7. COMMON MISTAKES TO AVOID

❌ Mistake 1: Using EDT instead of EST

All date/time stamps must be in EST year-round. Never use EDT. This is the most common error.

❌ Mistake 2: Inconsistent Extracted Date Time

Extracted Date Time must be identical across all files in the set. Different timestamps cause IR10 error.

❌ Mistake 3: Missing mandatory files

All 7 files must be in sync set. Even if File 02 has no data, include File Name Record + Header.

❌ Mistake 4: Framing Structure not at midnight

Framing Structure Start Date/Time must be midnight EST (HH:MM:SS = 00:00:00). Not following this breaks daily framing process.

❌ Mistake 5: Mismatched relationship start times

SDP-to-Meter and Meter-to-Comm relationships must have same Start Date/Time as Dials parameter.

❌ Mistake 6: Interval Length doesn't match meter

Interval Length in sync must match what meter actually reports. Mismatch causes meter read processing errors.

❌ Mistake 7: Dials doesn't match meter

Number of dials must match digits reported by meter. Incorrect dials causes VEE processing errors (Business Rule 25b).

❌ Mistake 8: Missing mandatory files

All 7 files must be in sync set. Missing files trigger IR10 report and entire set is rejected.

❌ Mistake 9: Reusing meter from another SDP

If meter was previously used at another SDP, old SDP-to-Meter relationship must be ended first. Business Rule 18 prevents meter from being active at multiple SDPs.

8. POST-SYNCHRONIZATION

Step 1: Monitor Reports

After submitting synchronization, monitor these reports in order:

IR10 - Incomplete Synchronization File Set Report

  • Generated immediately if file set is incomplete or incorrectly named
  • If you receive IR10, entire sync was rejected - fix issues and resubmit

IR14 - Staging Table Loader Exception Report

  • Check this first - contains file validation errors
  • Common issues: formatting, invalid codes, data type mismatches
  • If IR14 has errors, sync won't proceed to business rule validation

IR06 - Business Rule Exception Report

  • Lists business rule violations (if any)
  • Most common: Rule 18 (meter reuse), Rule 25 (dials mismatch), Rule 20 (relationship issues)

IR07 - Synchronization Processing Exception Report

  • Detailed processing exceptions
  • Review after IR06 to understand specific issues

Step 2: Verify in MDM/R GUI

  • Search for SDP ID in MDM/R web interface
  • Verify all elements are present and correct
  • Check relationship start dates/times
  • Confirm parameter values

Step 3: Verify Meter Reads Processing

  • Wait for first meter read submission from AMCC
  • Check that reads are being accepted and processed
  • Verify VEE processing is working correctly
  • Confirm billing quantities are being generated

9. VALIDATION CHECKLIST

Before Submission

  • All 7 files present with correct naming convention
  • Extracted Date Time identical in all files
  • Sequence number incremented from last sync
  • All date/time values in EST (not EDT)
  • File Name Record in every file
  • Header Record in every file

SDP Setup Validation

  • Universal SDP ID is unique and never used before
  • Premise address is complete and accurate
  • ESA ID provided (if applicable)

Meter Configuration Validation

  • Meter ID matches physical meter
  • Interval Length matches meter configuration
  • Channel Config Set appropriate for meter type
  • Number of Dials matches meter display
  • Install Status set to 'I'

Relationship Validation

  • SDP-to-Meter relationship has Start Date/Time, End is null
  • Meter-to-Comm relationship has Start Date/Time, End is null
  • Dials parameter Start Date/Time matches relationship starts
  • All relationship start times are same (synchronized)

Parameter Validation

  • VEE Service code is valid for customer class
  • Framing Structure appropriate for rate class
  • Framing Structure Start Date/Time at midnight EST
  • All mandatory parameters present

File Set Completeness

  • All 7 files present in manifest
  • File Name Records in all files
  • Header Records in all files
  • Extracted Date Time consistent across all files
  • Sequence Number incremented from previous sync

10. QUICK REFERENCE CHECKLIST

  • Verify Universal SDP ID is brand new (never synchronized before)
  • All date/time values in EST year-round
  • All 7 files present with File Name Record + Header
  • Extracted Date Time identical across all files
  • Meter configuration (interval, channel config, dials) matches physical meter
  • SDP-to-Meter and Meter-to-Comm relationships start at same time
  • Dials parameter starts at same time as relationships
  • Framing Structure starts at midnight EST
  • VEE Service appropriate for customer class
  • Billing Agent and AMI Operator relationships included
  • Communication module properly linked to meter
  • All mandatory elements present for billing quantities
